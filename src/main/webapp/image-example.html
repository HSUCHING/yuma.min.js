yuma.min.js

Welcome to the home of the yuma.min.js, the Open Source media annotation toolkit that makes content on your Web page annotatable with just a few lines of JavaScript! yuma.min.js works with

  • Images
  • Zoomable images
  • Audio

How Does yuma.min.js Work?

yuma.min.js makes it easy to attach annotation functionality to content on your Web pages. Simply include the yuma.min.js JavaScript file and CSS stylesheet, and you are ready to go. Developer documentation and code samples are available on our GitHub site. Give it a try! Click here (or on the 'Add Annotation' button) to add an annotation to the image below. Note that this demo will not store your annotations!

We are still getting things started and are hard at work bringing yuma.min.js into a stable state. If you encounter errors, unexpected behavior or generally weird stuff - please do contact us via our Google Group or post your issue on GitHub!

How Do I Store Annotations?

This demo page will not store annotations. If you want to make annotations on your site permanent, you need to set up an annotation storage server. Our upcoming sister project yuma4j will include everything you need to make this happen in a Java environment. (We're also working a hosting service which can be used instead of running your own server.)

About

yuma.min.js is based on the YUMA Universal Media Annotator, a proof-of-concept for social semantic annotation. YUMA was developed under the leadership of the Austrian Institute of Technology as part of the EuropeanaConnect research project. yuma.min.js picks out YUMA's best parts, packs them into a light user interface, and rolls everything into a toolbox that's flexible, configurable, and easy to re-use. Ultimately, we want you to be able to build your own, customized annotation tool with yuma.min.js with as little effort as possible - from small & simple to rich & complex.

Some of the YUMA features that are yet to come to yuma.min.js and yuma4j:

  • Tagging
  • Semantic Tagging (tagging based on controlled vocabularies)
  • RSS feeds
  • Sharing and embedding of annotations
  • Data interoperability and Linked Data publishing based on standard formats (OAC and W3C Media Fragments)
  • Support for HTML5 video
  • Customizable search and data export with different sort and filter options (including sort by position to support transcription use cases)
